# Playwright Demo Cursor And Zoom Polish Design

## Status

Approved for implementation planning.

## Goal

Improve the exported Playwright demo video polish with the smallest safe renderer changes: replace the square cursor marker with a realistic arrow cursor overlay, preserve existing click feedback controls, and make zoom ramps feel smoother without changing the `DemoProject` schema.

## Context

Current Tinker exports draw cursor telemetry in `packages/rendering/src/node/ffmpegFilterGraph.ts` with FFmpeg `drawbox`, so the mouse appears as a square. Generated Playwright projects in `packages/demo-assembly/src/compileProject.ts` also set `cursor: { clickEffect: "none" }`, so automatic demos suppress the default accent click ring. Zooms are produced from capture `zoomTarget` events and resolved through `packages/motion/src/cameraTransform.ts` using a quadratic easing ramp sampled frame-by-frame by the export path.

Recordly's implementation is useful as a design reference: it renders cursor images with hotspot anchors, shadows, click bounce/ring effects, and spring-smoothed cursor/camera motion. Recordly is AGPL-3.0, so this work must not copy source code or assets. We will only adopt the product ideas and implement them directly in Tinker's existing FFmpeg/export architecture.

## Non-Goals

- No `DemoProject` schema changes.
- No cursor style picker, cursor atlas, cursor-type telemetry, or system cursor capture.
- No Pixi/canvas renderer port from Recordly.
- No broad preview/editor UI work.
- No Recordly code or asset copying.
- No redesign of the browser capture event model.

## Approach

Use the fast export polish path.

1. Add a small deterministic cursor PNG generated by the Node export path from inline shape data, written into the render temp directory.
2. Replace plain cursor `drawbox` markers with FFmpeg image overlays using that cursor asset.
3. Position the cursor by its hotspot, not by its center, after mapping source coordinates into the padded output frame.
4. Keep click emphasis as generated FFmpeg geometry so existing `clickEffect` and `clickEffectDurationMs` behavior stays intact.
5. Improve zoom smoothness by replacing the quadratic `easeInOut` ramp in `cameraTransform.ts` with the smootherstep curve `6t^5 - 15t^4 + 10t^3` while preserving the existing per-frame sampling architecture.

## Cursor Rendering

The export graph should generate a transparent PNG arrow cursor with a subtle shadow in the render temp directory. Each cursor point maps through the existing `sourceToOutputPlacement` logic, then offsets by hotspot coordinates so the arrow tip lands on the recorded pointer position.

The export should still respect `resolveCursorSettings(project.cursor)`:

- `hidden: true` emits no cursor or click overlays.
- `clickEffect: "none"` shows the cursor but suppresses click emphasis.
- `clickEffect: "ring"` and `"ripple"` continue to show click emphasis for the resolved duration.

Generated Playwright demos should use the existing default click effect unless there is a specific reason to suppress it. If this is implemented by removing `cursor: { clickEffect: "none" }` from `compileProject`, it remains schema-compatible because absent cursor settings resolve to the documented defaults.

## Zoom Motion

Keep `normalizeZoomRegions`, `resolveDeterministicCameraTransform`, and `buildCameraIntervalsForExport` as the main pipeline. The first-pass polish should only adjust the easing curve used for non-linear zooms so the ramp has a softer start and finish than the current quadratic curve. This avoids introducing spring state into the export filter graph while improving perceived smoothness.

If this first pass is not enough, a later design can add true Recordly-style spring smoothing and motion blur as a separate, larger change.

## Data Flow

1. Browser capture records cursor, click, scroll, and zoom target events as it does today.
2. Demo assembly compiles those events into `cursorEvents` and `zooms` without adding new schema fields.
3. Rendering normalizes cursor telemetry and maps cursor positions into the output frame.
4. Rendering overlays the cursor image at the hotspot-adjusted position.
5. Rendering overlays click emphasis only when cursor settings allow it.
6. Rendering samples camera transforms frame-by-frame with the updated easing curve.

## Testing

Add or update tests to cover:

- The FFmpeg export graph uses a cursor image overlay for cursor points rather than plain cursor `drawbox` markers.
- Cursor placement accounts for source-to-output padding and hotspot alignment.
- `clickEffect: "none"` suppresses click emphasis while still rendering the cursor image.
- Default cursor settings render click emphasis when generated projects do not override them.
- The zoom easing regression test reflects the smoother early ramp while still reaching full zoom at the expected time.
- Existing real-export smoke tests still produce a playable MP4 with visible cursor and zoom output.

## Constraints

- Keep the change localized to rendering, motion easing, and generated project cursor defaults.
- Preserve deterministic output: no runtime font, browser, or OS cursor dependency.
- Prefer FFmpeg primitives already used by the renderer.
- Maintain preview/export intent parity for the existing cursor settings.
- Avoid unrelated refactors and formatter-only churn.
