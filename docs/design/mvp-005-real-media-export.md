# MVP-005 Real Media Export Design

## Goal

Replace the placeholder lavfi/drawbox export with an MP4 export that uses real project video assets, clip timing, camera motion, cursor/click effects, output dimensions, and ffprobe verification.

## Source Of Truth

- `docs/core-mvp-checklist.md` MVP-005
- `docs/dongha.md` export section
- `docs/architecture.md` rule: `DemoProject` is the source of truth; MP4 is only an artifact
- Existing asset preflight in `packages/rendering/src/node/assetResolution.ts`
- Existing motion-core behavior in `packages/editor/src/motion/*`

## Architecture

Use ffmpeg as the first production-fidelity renderer because the repo already has a safe argv-based ffmpeg runner, local asset preflight, and ffprobe verification. The renderer will build a deterministic filter graph:

```text
validated DemoProject snapshot
  -> asset preflight and path map
  -> real video inputs
  -> timeline media composition
  -> cursor/click overlay filters in source space
  -> camera transform filter
  -> h264 MP4
  -> ffprobe summary
```

The exporter must never shell-expand user strings. Every ffmpeg/ffprobe call stays `spawn(command, args)` with an argv array.

## Motion-Core Sharing

Rendering cannot import motion utilities from `@tinker/editor` because `@tinker/editor` already depends on `@tinker/rendering`. That would create a package cycle and make export depend on React/editor UI. The correct package boundary is a small shared motion package:

```text
@tinker/motion
  cursorTelemetry
  autoZoomSuggestions
  cameraTransform

@tinker/editor -> @tinker/motion
@tinker/rendering -> @tinker/motion
```

`@tinker/editor` will continue to re-export the motion APIs so existing UI code can keep importing from `@tinker/editor` until later cleanup.

## Renderer Strategy

For MVP-005, the ffmpeg renderer should prioritize correctness for demo timelines over general video-editor complexity:

- Clone and freeze the project at export start, then validate the render plan before ffmpeg starts.
- Resolve each required video asset once before invoking ffmpeg.
- Create one ffmpeg input per timeline clip so `sourceStart/sourceEnd` can be trimmed independently.
- Composite clips onto a generated background canvas in timeline time using `setpts=PTS-STARTPTS+clip.start/TB` plus `overlay=enable='between(t,start,end)'`.
- Scale/pad each clip to the export dimensions before timeline composition.
- Draw cursor/click effects onto the composed source-space media layer, matching preview's motion-layer model.
- Apply camera motion to the composed media plus cursor effects with a crop/scale stage derived from shared motion-core zoom normalization.
- Encode video-only MP4 with `libx264`, `yuv420p`, and `+faststart`.
- Probe the artifact and return the probe summary with path, duration, dimensions, and MIME metadata.

## Camera Scope

MVP-005 renders explicit zoom/camera keyframes from `project.zooms` and cursor/click overlays from `project.cursorEvents`. Export splits the composed timeline into static camera windows, resolves each window through shared motion-core target/focus/scale normalization, applies a fixed crop/scale per window, then concats the windows back to the original duration.

Animated zoom ramp/easing parity is intentionally deferred. The MVP-005 ffmpeg graph samples motion-core once per static camera interval; it does not generate per-frame crop/scale expressions for ramp-in, ramp-out, or easing curves. Stateful cursor-follow recentering also remains preview-only for MVP-005. MVP-009 owns frame-sampled animated ramp/easing, cursor-follow parity, and frame-level assertions for those transitions.

## Residual Risks

- Export output path policy is still intentionally loose in MVP-005: callers can choose any `.mp4` path accepted by the local process. MVP-007 owns restricting output paths to explicit user-selected or app-owned directories and blocking traversal-style output paths.

## Fixtures

The current `packages/project-schema/fixtures/assets/capture-001.mp4` is a tiny placeholder file. MVP-005 needs a valid, deterministic, very short MP4 fixture generated by ffmpeg. The fixture should be visual enough for smoke checks:

- left side red, right side blue, or a strong moving/positioned mark
- no audio
- small dimensions to keep repo/test cost low
- referenced by dedicated render tests rather than inflating every test runtime

## Error Handling

- Missing/unsafe assets still fail in preflight before ffmpeg starts.
- Invalid project JSON fails before export starts.
- Invalid output extension fails before export starts.
- ffmpeg failures return the command phase and stderr through a structured export error in MVP-006; MVP-005 keeps the current thrown error but must avoid partial success claims.
- ffprobe failures fail export verification.

## Acceptance Evidence

MVP-005 is complete only when current evidence proves:

- A one-clip fixture exports real visible source media.
- A trimmed fixture uses only the intended source segment.
- A zoom fixture changes the real media transform during the zoom window.
- A cursor/click fixture renders visible effects in the correct positions before and during camera motion.
- The returned result includes artifact path, duration, dimensions, and probe summary.
- `pnpm validate:schema`, `pnpm typecheck`, `pnpm -r test`, and `pnpm --filter @tinker/web build` pass.
